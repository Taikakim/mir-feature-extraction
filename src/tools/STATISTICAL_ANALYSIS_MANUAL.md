# Statistical Analysis Tool — Manual

Analyzes `.INFO` files generated by the MIR pipeline. Computes per-feature statistics, correlations, multicollinearity, dimensionality, feature clustering, and mutual information. Produces console output, JSON exports, and dark-themed PNG plots.

---

## Quick Start

```bash
# Basic statistics
python src/tools/statistical_analysis.py /path/to/data

# Full feature selection analysis with plots and JSON export
python src/tools/statistical_analysis.py /path/to/data \
  --feature-select --per-track -o stats.json --plots-dir ./plots

# Build track database + interactive explorer (do this once)
python src/tools/statistical_analysis.py /path/to/data \
  --build-db ./tracks.csv --plots-dir ./plots
# Then explore instantly in browser
xdg-open ./plots/feature_explorer.html
```

---

## Command Reference

```
statistical_analysis.py [path] [options]
```

### Required Argument

| Arg | Description |
|-----|-------------|
| `path` | Root directory to scan for `.INFO` files (recursive) |

> [!NOTE]
> `path` is optional only when using `--legend` (which prints a glossary and exits).

### Output Options

| Flag | Description |
|------|-------------|
| `-o, --output FILE` | Save all results to JSON |
| `-v, --verbose` | Show detailed per-feature output including outlier files |
| `--no-outliers` | Strip outlier details from output and JSON |
| `--plots-dir DIR` | Directory for PNG plots (default: same dir as `--output`, or cwd) |

### Data Aggregation

| Flag | Description |
|------|-------------|
| `--per-track` | Aggregate all crops per track into a single entry. Numeric features are **averaged** (only over crops that have that key — missing keys don't produce zeros). Categorical features use the most common value. |

### Top-N Query

Fast lookup mode — skips full statistics, just loads and ranks.

| Flag | Description |
|------|-------------|
| `--top N` | Show top N values for a feature (requires `--key`) |
| `--key KEY` | Which feature to query (case-insensitive, suggests matches on typo) |
| `--bottom` | Reverse sort: show bottom N instead of top N |

```bash
# Most popular tracks (one per track)
python src/tools/statistical_analysis.py /path/to/data --top 20 --key popularity --per-track

# Quietest crops by LUFS
python src/tools/statistical_analysis.py /path/to/data --top 10 --key lufs --bottom

# Fastest BPM (all crops, not aggregated)
python src/tools/statistical_analysis.py /path/to/data --top 5 --key bpm
```

### Correlation Analysis

| Flag | Description |
|------|-------------|
| `-c, --correlation` | Compute pairwise Pearson correlations |
| `--corr-threshold F` | Threshold for flagging high correlations (default: 0.7) |

### Feature Selection Analyses

| Flag | Description |
|------|-------------|
| `--feature-select` | **Meta-flag**: runs all five below + recommendation engine |
| `--vif` | Variance Inflation Factor — detects multicollinearity |
| `--pca` | PCA — effective dimensionality and feature loadings |
| `--cluster` | Hierarchical clustering on correlation distance |
| `--mi` | Mutual information — captures non-linear dependencies |
| `--cluster-threshold F` | Distance cutoff for clustering (default: 0.5, range 0–1) |

### Scatter & Quadrant Plots

| Flag | Description |
|------|-------------|
| `--scatter KEY_X KEY_Y` | Interactive X/Y scatter of two features (auto-aggregates per track) |
| `--quadrant X+ X- Y+ Y-` | Four-pole quadrant plot with bipolar axes (auto-aggregates per track) |

Both produce interactive HTML files (plotly) with hover tooltips showing track names and values. A static PNG is also saved if `kaleido` is installed.

```bash
# BPM vs LUFS scatter
python src/tools/statistical_analysis.py /path/to/data --scatter bpm lufs --plots-dir ./plots

# Quadrant: brightness vs spectral_centroid on X, lufs vs spectral_flatness on Y
python src/tools/statistical_analysis.py /path/to/data \
  --quadrant brightness spectral_centroid_mean lufs spectral_flatness_mean --plots-dir ./plots
```

### Track Database & Interactive Explorer

| Flag | Description |
|------|-------------|
| `--build-db FILE` | Scan .INFO files once, aggregate per track, save CSV database |
| `--explorer DB_FILE` | Generate interactive HTML explorer from pre-built CSV (instant, no path needed) |

The explorer is a self-contained HTML file with:
- **Scatter** and **Quadrant** mode toggle
- Dropdown selectors for any feature pair
- Mouseover tooltips with track name and values
- Trend line with correlation coefficient

```bash
# Build database (slow — scans all .INFO files, only do once)
python src/tools/statistical_analysis.py /path/to/data --build-db ./tracks.csv --plots-dir ./plots

# Regenerate explorer instantly from existing database
python src/tools/statistical_analysis.py --explorer ./tracks.csv --plots-dir ./plots

# Open in browser
xdg-open ./plots/feature_explorer.html
```

> [!TIP]
> The explorer embeds all data in the HTML file (~5MB for ~4500 tracks × 78 features). Switching axes is instant — no server or re-computation needed.

### Information

| Flag | Description |
|------|-------------|
| `-l, --legend` | Print glossary of all statistical terms and exit |
| `-h, --help` | Show usage and exit |

## Analyses Explained

### Basic Statistics

Always computed. For each numeric feature: count, mean, median, std, min, max, Q1, Q3, IQR, outlier count (1.5×IQR method), and outlier file names.

For categorical features: value counts and top values.

---

### Correlation (`--correlation`)

Pairwise Pearson correlation (r) between all numeric features. Outputs per-feature max |r| and which feature it's most correlated with.

| |r| range | Interpretation |
|-----------|----------------|
| 0.0 – 0.3 | Weak — independent |
| 0.3 – 0.5 | Moderate — some shared info |
| 0.5 – 0.7 | Strong — consider if both needed |
| 0.7 – 0.9 | Very strong — likely redundant |
| 0.9 – 1.0 | Near identical — drop one |

**Plot**: `features_correlation_heatmap.png` — clustered heatmap reordered by hierarchical similarity.

---

### VIF (`--vif`)

Variance Inflation Factor measures how well each feature can be predicted from **all other features combined** (multicollinearity).

`VIF = 1 / (1 - R²)` where R² comes from regressing one feature on all others.

| VIF | Status |
|-----|--------|
| < 5 | ✅ OK |
| 5 – 10 | ⚠️ Moderate multicollinearity |
| > 10 | 🔴 Severe — feature is linearly predictable, drop it |
| ∞ | Feature is a perfect linear combination of others |

**Plot**: `features_vif_chart.png` — horizontal bar chart with threshold lines at 5 and 10.

---

### PCA (`--pca`)

Principal Component Analysis finds the effective dimensionality — how many independent axes of variation exist in your feature space.

Key outputs:
- **Components for 90%/95% variance** — how many PCs you actually need
- **Effective dimensionality ratio** — fraction of features carrying real information
- **Top loadings per component** — which features drive each axis

**Plots**:
- `features_pca_scree.png` — per-component variance and cumulative curve
- `features_pca_loadings.png` — heatmap of feature loadings on top components

---

### Hierarchical Clustering (`--cluster`)

Groups features by correlation distance (`1 - |r|`). Features within the same cluster are redundant — keep only the **representative** (the one with lowest mean correlation to everything else).

`--cluster-threshold` controls strictness:
- **0.3** = strict (only very highly correlated features grouped)
- **0.5** = default (moderate correlation grouped)
- **0.7** = loose (weakly correlated features may be grouped)

**Plot**: `features_dendrogram.png` — tree diagram showing feature relationships.

---

### Mutual Information (`--mi`)

Captures **non-linear** dependencies that Pearson correlation misses. Computed pairwise using sklearn's `mutual_info_regression`.

The summary highlights pairs with high MI but low |r| — these have non-linear relationships invisible to correlation analysis.

**Plot**: `features_mi_heatmap.png` — clustered MI heatmap.

> [!NOTE]
> MI is the slowest analysis (~45s for 57 features × 5000 samples). It scales as O(n²) in feature count.

---

### Recommendation Engine (`--feature-select`)

Combines all analyses to suggest a final feature set:

1. **Drop** features with VIF > 10 (severe multicollinearity)
2. **Drop** redundant cluster members (keep representative only)
3. **Warn** about non-linear entanglements from MI
4. **Report** effective dimensionality from PCA

Output is a concrete keep/drop list with reasons for each dropped feature.

---

## Output Files

### JSON (`--output`)

Contains all computed results in a single file:

| JSON key | Contents |
|----------|----------|
| `feature_name` | Basic stats (count, mean, std, etc.) |
| `_correlation_analysis` | Per-feature max correlations |
| `_vif` | Per-feature VIF and R² |
| `_pca` | Components, variance ratios, loadings |
| `_clusters` | Cluster assignments and representatives |
| `_mutual_information` | Top MI pairs, feature names |
| `_recommended_features` | Keep/drop lists with reasons |
| `_legend` | Glossary of all keys |

### Plots (`--plots-dir`)

| File | Generated by |
|------|-------------|
| `features_correlation_heatmap.png` | `--correlation` |
| `features_vif_chart.png` | `--vif` |
| `features_pca_scree.png` | `--pca` |
| `features_pca_loadings.png` | `--pca` |
| `features_dendrogram.png` | `--cluster` |
| `features_mi_heatmap.png` | `--mi` |
| `scatter_X_vs_Y.html` / `.png` | `--scatter` |
| `quadrant_Xp_Xn_Yp_Yn.html` / `.png` | `--quadrant` |
| `tracks.csv` | `--build-db` |
| `feature_explorer.html` | `--build-db` or `--explorer` |

## Common Recipes

```bash
# ─── Full analysis (one value per track, all plots) ───
python src/tools/statistical_analysis.py /path/to/data \
  --feature-select --per-track --no-outliers \
  -o stats.json --plots-dir ./plots

# ─── Just check which features are redundant ───
python src/tools/statistical_analysis.py /path/to/data --vif --cluster

# ─── Quick ranking queries ───
python src/tools/statistical_analysis.py /path/to/data --top 20 --key bpm --per-track
python src/tools/statistical_analysis.py /path/to/data --top 10 --key lufs --bottom
python src/tools/statistical_analysis.py /path/to/data --top 5 --key danceability --per-track

# ─── Correlation only, stricter threshold ───
python src/tools/statistical_analysis.py /path/to/data --correlation --corr-threshold 0.5

# ─── Verbose basic stats with outlier files ───
python src/tools/statistical_analysis.py /path/to/data --verbose

# ─── Print glossary ───
python src/tools/statistical_analysis.py --legend
```
